# 기존 저장소 가져오기

이미 GitHub에 백업이 있다면 새로 만들지 않고 그대로 가져올 수 있습니다.

## 이런 경우에 사용하세요

- 다른 PC에서 쓰던 백업을 이어서 쓰고 싶을 때
- 이미 GitHub에 올려 둔 폴더를 새 컴퓨터로 가져올 때

## 가져오기 순서

1. `기존 백업 가져오기` 또는 상단 `가져오기`를 누릅니다.
2. `내 저장소 불러오기`를 눌러 목록을 가져오거나 주소를 직접 입력합니다.
3. 가져올 저장소를 선택합니다.
4. `폴더 선택`을 눌러 저장 위치를 고릅니다.
5. 원하면 `가져오기가 끝나면 바로 폴더 감시 시작`을 켭니다.
6. `가져오기 시작`을 누릅니다.

## 저장 위치는 어떻게 고르면 좋나요

비어 있는 폴더를 고르는 것이 가장 안전합니다.

이미 다른 파일이 많은 폴더는 가져오기에 실패할 수 있습니다.

## 주소를 직접 입력할 때

아래 두 형식 중 하나를 사용할 수 있습니다.

- `https://github.com/owner/repository`
- `owner/repository`

## 가져오기가 끝나면

- 대시보드에서 연결된 저장소를 확인할 수 있습니다.
- 감시 시작 옵션을 켰다면 바로 자동 동기화가 준비됩니다.

## 이미 GitHub 연결이 있는 폴더를 고른 경우

대시보드에서 폴더를 선택했을 때 기존 GitHub 연결을 자동으로 찾을 수 있습니다.

다만 GitHub 연결 주소가 `git@github.com:` 형식이면 자동 백업이 잘 되지 않을 수 있습니다.

이 경우에는 `https://github.com/owner/repository.git` 형식으로 바꿔 주세요.

## 자주 생기는 문제

- 저장 위치가 비어 있지 않음
- 다른 저장소와 연결된 폴더를 선택함
- 로그인되지 않음
- 인터넷 연결이 불안정함
- 기존 저장소 연결이 SSH 형식임

이 경우에는 다른 폴더를 선택하거나 다시 로그인한 뒤 다시 시도하세요.
