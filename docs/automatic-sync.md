# 자동 동기화 안내

GitBackup은 폴더를 계속 지켜보다가 변경이 생기면 자동으로 동기화합니다.

## 자동 동기화는 이렇게 동작합니다

1. 폴더 감시를 시작합니다.
2. 파일 추가, 수정, 삭제 같은 변화를 찾습니다.
3. 잠시 기다리며 변경을 한 번에 모읍니다.
4. 준비가 되면 GitHub에 올립니다.

즉, 파일을 저장할 때마다 바로 여러 번 올리지 않고 잠깐 묶어서 처리합니다.

## 감시 서비스 카드에서 볼 수 있는 것

대시보드 아래쪽 `감시 서비스` 카드에서 아래 내용을 확인할 수 있습니다.

- 현재 상태
- 로그인, 폴더 선택, 저장소 연결, 감시 및 동기화 단계
- 대기 중인 변경 수
- 최근 동기화 시간
- 최근 백업 기록
- 최근 처리 안내

## 상태 표시 읽는 법

- `대기 중`: 아직 다음 단계가 시작되지 않음
- `진행 중`: 지금 이 단계를 처리하고 있음
- `완료`: 이 단계가 끝났음
- `오류`: 확인이 필요한 문제가 있음

## 왜 잠시 기다리나요

문서를 저장할 때 프로그램이 여러 파일을 연달아 바꾸는 경우가 있습니다.

GitBackup은 이런 상황에서 너무 자주 동기화하지 않도록 잠시 기다립니다.

## 지금 바로 올리고 싶을 때

`지금 동기화` 버튼을 누르면 대기 중인 변경 내용을 바로 올릴 수 있습니다.

## 자동 동기화를 끄고 싶을 때

두 가지 방법이 있습니다.

- 대시보드에서 `감시 중지`
- 설정에서 `자동 동기화` 끄기

## 큰 파일 안내

100MB가 넘는 파일은 자동 업로드가 실패할 수 있습니다.

앱에서 큰 파일 경고가 보이면 파일 크기를 먼저 확인하세요.
