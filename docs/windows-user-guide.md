# Windows 설치 및 사용 가이드

이 문서는 GitBackup을 처음 다운로드한 사람이 설치부터 첫 백업까지 바로 진행할 수 있도록 만든 안내입니다.

## 1. 준비할 것

- Windows PC
- 인터넷 연결
- GitHub 계정

## 2. 설치 파일 다운로드

배포 페이지에서 GitBackup Windows 설치 파일 `.exe`를 다운로드합니다.

파일 이름은 버전에 따라 다를 수 있습니다.

## 3. 설치하기

1. 다운로드한 `.exe` 파일을 실행합니다.
2. 설치 위치를 확인합니다.
3. `설치`를 진행합니다.
4. 설치가 끝나면 GitBackup을 실행합니다.

Windows에서 경고가 보이면 먼저 파일이 공식 배포본인지 확인하세요.

## 4. 첫 실행 후 가장 먼저 할 일

앱이 열리면 `GitHub 로그인`을 누릅니다.

브라우저가 열리면 GitHub 로그인과 연결 승인을 마칩니다.

앱 화면에 코드가 보이면 브라우저에서 그대로 입력하면 됩니다.

## 5. 새 백업 시작하기

새로 백업을 만들고 싶다면 아래 순서로 진행하세요.

1. `새 백업 감시`를 누릅니다.
2. 백업할 폴더를 선택합니다.
3. 저장소 이름을 입력합니다.
4. `새 비공개 저장소 만들기`를 누릅니다.
5. `감시 시작`을 누릅니다.

이제 파일을 바꾸면 GitBackup이 자동으로 변경을 감지하고 GitHub에 올립니다.

## 6. 기존 백업 가져오기

이미 GitHub에 백업이 있다면 아래 순서로 진행하세요.

1. `기존 백업 가져오기`를 누릅니다.
2. 저장소를 선택하거나 주소를 입력합니다.
3. 비어 있는 폴더를 저장 위치로 선택합니다.
4. 필요하면 `가져오기가 끝나면 바로 폴더 감시 시작`을 켭니다.
5. `가져오기 시작`을 누릅니다.

## 7. 대시보드에서 확인할 것

대시보드에서는 아래 내용을 확인하면 됩니다.

- `GitHub 연결`: 로그인 상태
- `연결 저장소`: 현재 백업 대상
- `감시 서비스`: 지금 어느 단계인지
- `오류 안내`: 다음에 해야 할 행동

## 8. 자주 쓰는 버튼

- `GitHub 로그인`: 계정 연결
- `새 백업 감시`: 새 폴더 백업 시작
- `기존 백업 가져오기`: 이미 있는 백업 연결
- `감시 시작`: 자동 감시 켜기
- `감시 중지`: 자동 감시 끄기
- `지금 동기화`: 지금 바로 백업 보내기
- `로그아웃`: 계정 연결 해제

## 9. 사용 중 기억하면 좋은 점

- 로그인하지 않으면 백업 기능을 사용할 수 없습니다.
- 100MB가 넘는 파일은 자동 업로드가 어려울 수 있습니다.
- 기존 저장소 연결은 HTTPS 주소가 더 안전합니다.
- 인터넷이 느리면 동기화에 시간이 걸릴 수 있습니다.
