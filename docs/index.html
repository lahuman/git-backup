왜 GitBackup인가요?

복잡한 백업 대신,
누구나 이해할 수 있는 흐름으로

1

어렵지 않은 시작

Git을 몰라도 괜찮습니다. 앱을 열고 로그인한 뒤, 백업할 폴더를 고르면 됩니다.

2

자동 감시와 빠른 반영

파일이 바뀌었을 때 바로 반영하거나, 자동 감시로 편하게 백업 흐름을 유지할 수 있습니다.

3

기존 백업도 가져오기 가능

이미 GitHub에 올려둔 저장소가 있다면 가져오기 기능으로 내 PC에 다시 연결할 수 있습니다.

4

일반 사용자 중심 UI

복잡한 명령어 대신 버튼과 화면 안내로 진행할 수 있어, 윈도우 사용자에게 익숙한 흐름을 제공합니다.

5

큰 파일도 미리 확인

업로드 전에 큰 파일을 먼저 확인해 불필요한 실패나 혼란을 줄이는 데 도움을 줍니다.

6

개인 백업용으로 딱 맞는 구성

문서, 작업파일, 개인 프로젝트 폴더처럼 자주 바뀌는 자료를 간단히 정리하고 싶을 때 적합합니다.

사용 방법

3단계로 시작하는 파일 백업

STEP 1

앱 설치 후 GitHub 로그인

복잡한 설정 없이 계정을 연결해 백업 준비를 시작합니다.

STEP 2

백업할 폴더 선택

보관하고 싶은 문서 폴더나 작업 폴더를 선택합니다.

STEP 3

감시 시작 또는 바로 동기화

자동 감시로 편하게 쓰거나, 필요할 때 직접 동기화할 수 있습니다.

메인 화면

현재 연결된 계정, 관리 중인 폴더, 동기화 버튼, 최근 상태를 쉽게 볼 수 있습니다.

설정

자동 동기화도 쉽게 설정

동기화 간격, 기본 저장소 공개 범위, 큰 파일 알림 같은 옵션을 단순하게 조절할 수 있습니다.

가져오기

기존 저장소 다시 연결하기

이미 GitHub에 있는 백업 저장소를 내 컴퓨터로 가져와 이어서 관리할 수 있습니다.

FAQ

자주 묻는 질문

Git을 잘 몰라도 사용할 수 있나요?

네. Git 명령을 직접 다루는 대신, 로그인과 버튼 중심 흐름으로 백업을 시작할 수 있게 구성했습니다.

누구를 위한 앱인가요?

개발자뿐 아니라, 중요한 폴더를 쉽고 빠르게 백업하고 싶은 일반 Windows 사용자를 위한 앱입니다.

GitHub 계정이 꼭 필요한가요?

네. GitBackup은 GitHub 저장소에 파일을 보관하는 방식이므로 GitHub 계정 연결이 필요합니다.

기존에 GitHub에 올려둔 백업도 다시 가져올 수 있나요?

가능합니다. 가져오기 화면에서 저장소를 불러오거나 주소를 입력해 연결할 수 있습니다.

Windows에서만 사용 가능한가요?

이 페이지는 Windows 사용자를 중심으로 소개하고 있으며, 다운로드 안내도 Windows 기준으로 구성되어 있습니다.
