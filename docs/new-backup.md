# 새 백업 시작하기

이 문서는 처음으로 폴더 백업을 시작하는 방법을 안내합니다.

## 시작 전에 준비할 것

- GitHub 로그인 완료
- 백업할 폴더 확인
- 인터넷 연결 확인

## 새 백업 시작 순서

1. 대시보드에서 `새 백업 감시`를 누릅니다.
2. 백업할 폴더를 선택합니다.
3. 연결된 저장소가 없으면 저장소 이름을 입력합니다.
4. `새 비공개 저장소 만들기`를 누릅니다.
5. 저장소가 연결되면 `감시 시작`을 누릅니다.

필요하면 `지금 동기화`를 눌러 바로 한 번 올릴 수도 있습니다.

## 폴더를 고를 때 팁

- 자주 바뀌는 작업 폴더를 고르면 편합니다.
- 너무 넓은 상위 폴더 전체를 고르면 처음 동기화가 오래 걸릴 수 있습니다.
- 임시 파일이 많은 폴더는 먼저 정리하면 좋습니다.

## 폴더를 고른 뒤 일어나는 일

- 이미 GitHub 저장소와 연결된 폴더면 연결 저장소를 자동으로 찾을 수 있습니다.
- 연결된 저장소가 없으면 새 저장소를 만들거나 기존 저장소를 가져오면 됩니다.
- 준비가 끝나면 감시와 동기화를 시작할 수 있습니다.

## 시작 후 앱이 하는 일

GitBackup은 선택한 폴더를 지켜보다가 변경이 생기면 잠시 기다린 뒤 한 번에 동기화합니다.

여러 파일을 연달아 저장해도 바로 여러 번 올리지 않도록 설계되어 있습니다.

## 바로 확인하는 방법

아래 항목이 보이면 준비가 끝난 것입니다.

- 선택 폴더가 표시됨
- 연결 저장소가 표시됨
- 감시 서비스 카드에 현재 단계가 표시됨

## 주의할 점

- 100MB가 넘는 파일은 경고가 표시될 수 있습니다.
- 로그인하지 않은 상태에서는 새 백업을 시작할 수 없습니다.
- 기존 SSH 저장소 주소는 자동 백업에 바로 쓰기 어려우므로 HTTPS 연결이 더 안전합니다.
