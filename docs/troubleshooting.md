# 문제 해결

잘 되지 않을 때는 아래 순서대로 확인해 보세요.

## GitHub 로그인이 되지 않아요

- 인터넷이 연결되어 있는지 확인하세요.
- 브라우저에서 로그인과 승인까지 끝냈는지 확인하세요.
- 코드가 만료되었으면 새 코드를 받으세요.
- 앱을 완전히 닫은 뒤 다시 열어 보세요.

## 로그인 전에는 버튼이 눌리지 않아요

정상입니다.

GitBackup은 로그인 전에는 폴더 선택, 저장소 가져오기, 동기화, 감시 시작을 막아 둡니다.

먼저 `GitHub 로그인`을 완료해 주세요.

## 기존 저장소가 있는데 연결 저장소가 없다고 나와요

먼저 대시보드에서 폴더를 다시 선택해 보세요.

GitBackup은 기존 GitHub 연결을 자동으로 찾을 수 있습니다.

그래도 안 되면 아래를 확인하세요.

- 해당 폴더가 실제 GitHub 저장소와 연결되어 있는지
- 연결 주소가 `https://github.com/...` 형식인지
- 다른 저장소와 연결된 폴더가 아닌지

## SSH 저장소 주소 때문에 진행되지 않아요

현재는 HTTPS 연결이 가장 안전합니다.

아래 형식으로 바꿔 주세요.

- `https://github.com/owner/repository.git`

아래 형식은 자동 백업에서 바로 쓰기 어렵습니다.

- `git@github.com:owner/repository.git`

## 동기화 중 시간이 오래 걸리거나 timeout이 나와요

- 인터넷 연결을 확인하세요.
- 회사나 학교 네트워크를 쓰고 있으면 다른 네트워크에서도 시도해 보세요.
- VPN이나 보안 프로그램을 잠시 끄고 다시 시도해 보세요.
- `지금 동기화`를 다시 눌러 보세요.

현재 앱은 GitHub 응답이 오래 걸리면 안내 메시지를 보여 줍니다.

## 저장소 목록을 불러오지 못해요

- GitHub 로그인이 유지되어 있는지 확인하세요.
- 잠시 후 다시 시도하세요.
- GitHub 쪽 연결이 일시적으로 느릴 수 있습니다.

## 가져오기가 시작되지 않아요

- 저장소를 선택했는지 확인하세요.
- 저장 위치를 선택했는지 확인하세요.
- 저장 위치가 비어 있는 폴더인지 확인하세요.

## 자동 동기화가 되지 않아요

- `감시 중` 상태인지 확인하세요.
- 로그인 상태가 유지되는지 확인하세요.
- `지금 동기화`를 눌러 수동으로 한 번 확인해 보세요.
- 큰 파일 경고가 있는지 확인하세요.
- `감시 서비스` 카드에 어느 단계에서 멈췄는지 확인하세요.

## 큰 파일 경고가 나와요

100MB가 넘는 파일은 일반적인 업로드에 적합하지 않을 수 있습니다.

다음 중 하나를 선택하세요.

- 파일 크기를 줄입니다.
- 해당 파일을 백업 폴더 밖으로 옮깁니다.
- 정말 필요한 파일인지 다시 확인합니다.

## 설치는 되었는데 Windows 경고가 보여요

먼저 다운로드한 파일이 공식 배포본인지 확인하세요.

공식 배포본이 맞다면 경고 화면에서 추가 정보를 확인한 뒤 실행할 수 있습니다.

## 로그 파일은 어디에 있나요

설치형 앱에서는 보통 아래 위치에서 로그를 확인할 수 있습니다.

- `%APPDATA%\GitBackup\logs\main.log`

## 계속 안 될 때

문제가 생긴 화면과 버튼 이름을 메모해 두면 도움을 받기 쉽습니다.

가능하면 아래 정보를 함께 정리하세요.

- 어떤 화면에서 문제가 생겼는지
- 어떤 버튼을 눌렀는지
- 어떤 문구가 보였는지
- 인터넷이 연결되어 있었는지
- 로그 파일에 어떤 메시지가 남았는지
